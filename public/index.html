Security Hacking Lab

Your playground to test real-world cybersecurity vulnerabilities

Welcome to the Hacking Lab

This lab provides a safe environment for penetration testers and security enthusiasts to simulate and practice various web application vulnerabilities, weak SSH configurations, and database exploits. Set up in a Docker environment, it's perfect for hands-on learning.

Get Started

Lab Features

Vulnerable Web App

The lab includes a web application with common vulnerabilities such as SQL injection, XSS, file inclusion, and more. Perfect for testing web attacks.

SSH Brute-force Testing

A vulnerable SSH service is available, allowing you to practice brute-force attacks and testing weak configurations.

Database Exploits

MariaDB is included, providing a backend for database-related vulnerabilities such as SQL injection, misconfigurations, and privilege escalation.

How to Use the Hacking Lab

  1. Set up Docker: Ensure Docker is installed on your machine. If not, download and install Docker.
  2. Clone the lab repository: Run the command below to clone the lab repository: 
    git clone https://github.com/your-username/security-hack-lab.git
  3. Run the Docker containers: Use the following command to start the hacking lab: 
    docker-compose up -d
  4. Access the web app: Open your browser and navigate to http://localhost:8080 to interact with the vulnerable web app.
  5. Use your hacking tools: You can now use tools like Burp Suite, OWASP ZAP, or SQLMap to test for vulnerabilities and exploit them.

Tools to Use with the Lab

Burp Suite

A comprehensive tool for web vulnerability scanning, intercepting requests, and automated attack simulations.

SQLMap

A powerful open-source tool for automating SQL injection attacks and database takeover.

OWASP ZAP

A free security tool for finding vulnerabilities in web applications and testing for XSS, CSRF, and other attacks.
